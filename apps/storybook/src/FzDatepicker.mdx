# FzDatepicker

Date and date-range picker for the Fiscozen design system, built on top of [VueDatePicker v12](https://vue3datepicker.com/). Provides Fiscozen-branded styling, `FzInput` integration, and a custom time-picker overlay.

## Features

- Single date, date range, week, month, and year selection
- Inline time picker with hours, minutes, and optional seconds
- Calendar menu teleported to `<body>` by default (works inside modals/dialogs)
- Keyboard navigation and text input support
- Customizable placement via Floating UI
- Legacy v8 prop compatibility (transparent remapping)
- Responsive mobile layout

## Installation

```bash
npm install @fiscozen/datepicker
```

## Basic Usage

```vue
<script setup>
import { ref } from 'vue'
import { FzDatepicker } from '@fiscozen/datepicker'

const date = ref()
</script>

<template>
  <FzDatepicker
    v-model="date"
    :inputProps="{ label: 'Select a date', placeholder: 'dd/mm/yyyy' }"
  />
</template>
```

## States

### Required

Required datepickers show an asterisk (\*) next to the label and set `aria-required` on the visible `<input>`.

```vue
<FzDatepicker
  v-model="date"
  :inputProps="{ label: 'Birth date', required: true }"
/>
```

### Disabled

A disabled datepicker is non-interactive (calendar won't open, input is read-only). Useful in conditional forms.

```vue
<FzDatepicker
  v-model="date"
  disabled
  :inputProps="{ label: 'Disabled date' }"
/>
```

### Error

Pass `error: true` via `inputProps` to enable the error styling and the `errorMessage` slot. The slot content is rendered with `role="alert"` and linked to the input via `aria-describedby`.

```vue
<FzDatepicker
  v-model="date"
  :inputProps="{ label: 'Date', error: hasError }"
>
  <template #errorMessage>Please choose a valid date</template>
</FzDatepicker>
```

### Help text

The `helpText` slot is shown only when there is no active error. It is linked to the input via `aria-describedby`.

```vue
<FzDatepicker
  v-model="date"
  :inputProps="{ label: 'Birth date' }"
>
  <template #helpText>Format: dd/mm/yyyy</template>
</FzDatepicker>
```

### Clearable

Adding `clearable` shows a ✕ button when the input has a value. The ✕ icon comes from the inner `FzInput` (the DS overrides VueDatePicker's own `#clear-icon` to empty to avoid duplicates).

```vue
<FzDatepicker
  v-model="date"
  clearable
  :inputProps="{ label: 'Booking date' }"
  @fzdatepicker:clear="onCleared"
/>
```

### Time Picker in Overlay Mode (with branded clock/calendar toggles)

When `timePickerInline: false` the time picker opens as an overlay, with a clock-icon button to enter it and a calendar-icon button to return. Both icons are pre-filled with `FzIcon` (slot `clock-icon` / `calendar-icon`).

```vue
<template>
  <FzDatepicker
    v-model="date"
    :enableTimePicker="true"
    :timePickerInline="false"
    :enableMinutes="true"
    :autoApply="false"
    :inputProps="{ label: 'Date + time' }"
  />
</template>
```

### Confirm / Cancel buttons (`autoApply: false`)

Setting `autoApply: false` unlocks the DS-provided action row (Cancella / Seleziona). The buttons inherit `inputProps.environment`.

```vue
<template>
  <FzDatepicker
    v-model="date"
    :autoApply="false"
    :inputProps="{ label: 'Choose date', environment: 'backoffice' }"
  />
</template>
```

### Form with name attribute (recommended for new code)

Use `inputAttrs.name` to set the input's `name` attribute so form helpers (e.g. `FormData`, jQuery `$(form).find("[name]")`, native form submission) can pick the field up. Note: if you already pass a top-level `name` prop, that wins — see Best Practices below.

```vue
<template>
  <form @submit.prevent="onSubmit">
    <FzDatepicker
      v-model="businessStart"
      :inputAttrs="{ name: 'business_start' }"
      :inputProps="{ label: 'Inizio attività' }"
    />
    <button type="submit">Salva</button>
  </form>
</template>
```

## Accessibility

- Full keyboard navigation support via `arrowNavigation` (enabled by default)
- Text input support for typing dates directly
- Calendar popup is focusable and supports arrow key navigation
- Escape key closes the calendar popup
- Tab key moves focus away from the datepicker
- ARIA attributes are managed by the underlying `FzInput` component (`aria-invalid`, `aria-disabled`, `aria-required`, `aria-labelledby`, `aria-describedby`)
- Error messages are rendered with `role="alert"` and linked to the input via `aria-describedby`
- Help text is linked to the input via `aria-describedby` when no error is active

## Best Practices

- Always provide a `label` via `inputProps` for accessibility
- Use `placement` to control where the calendar appears relative to the input
- The `teleport` prop defaults to `"body"` which is the recommended setting, especially inside modals or dialogs
- Use `modelType` when you need the emitted value in a specific string format (e.g. `'yyyy-MM-dd'`)
- Use `valueFormat` when you want to format the emitted value with date-fns before emission
- Set `autoApply: false` when using time picker or multi-date selection to give users a confirm/cancel step (also unlocks the Cancella/Seleziona buttons through the `#action-buttons` slot)
- For forms, set the `name` attribute via `:inputAttrs="{ name: '...' }"` so the datepicker value is included in form submissions. The top-level `name` prop is `@deprecated` (will be removed in v4.0.0) — see "Form with name attribute" example

## Related Components

- **FzInput**: The inner input element used by the datepicker
- **FzButton / FzIconButton**: Used for calendar navigation arrows and action buttons
- **FzIcon**: Used for the inline time-picker chevrons and the clock/calendar toggle icons
