# FzPagination

Page-based navigation control for paginated data sets. Provides intuitive page navigation with support for two-way binding, configurable labels, ellipsis handling, and responsive behavior.

## Features

- Two-way binding via v-model:currentPage for current page
- Configurable total number of pages
- Automatic URL sync with query parameters (enabled by default)
- Responsive layout (icon-only prev/next on mobile)
- Accessible navigation with semantic HTML and ARIA attributes
- Frontoffice and backoffice environment support

## Installation

```bash
npm install @fiscozen/pagination
```

## Basic Usage

```vue
<script setup>
import { FzPagination } from '@fiscozen/pagination'
import { ref } from 'vue'

const currentPage = ref(1)
</script>

<template>
  <FzPagination v-model:currentPage="currentPage" :totalPages="10" />
</template>
```

## URL Sync

URL sync is **enabled by default**. The component reads the initial page from `?page=` in the URL on mount and writes page changes back automatically.

### Setup (once, at app level)

Provide the Vue Router route so all FzPagination instances (even nested inside other components) can access it without prop drilling:

```vue
<script setup>
import { provideQueryStringRoute } from '@fiscozen/composables'
import { useRoute } from 'vue-router'

provideQueryStringRoute(useRoute())
</script>
```

If `provideQueryStringRoute()` is not called, the component falls back to the browser History API directly. This works in any environment (standalone pages, micro-frontends, non-Vue-Router apps).

### Initializing currentPage from the URL

Use `getInitialPageFromUrl` to set the correct page **before** the first API request. Without it, the ref would start at the default value and FzPagination would emit a corrected `update:currentPage` on mount, causing a redundant request.

```vue
<script setup>
import { FzPagination, getInitialPageFromUrl } from '@fiscozen/pagination'
import { ref } from 'vue'

const currentPage = ref(getInitialPageFromUrl())

// fetchData(currentPage.value) — first request already uses the correct page
</script>

<template>
  <FzPagination v-model:currentPage="currentPage" :totalPages="20" />
</template>
```

`getInitialPageFromUrl(defaultValue = 1, urlKey = 'page')` accepts an optional default and a custom query parameter key:

```ts
const page = getInitialPageFromUrl(0, 'p') // reads ?p=, defaults to 0
```

### Default behavior

```vue
<script setup>
import { FzPagination } from '@fiscozen/pagination'
import { ref } from 'vue'

const currentPage = ref(1)
</script>

<template>
  <!-- Syncs with ?page= by default -->
  <FzPagination v-model:currentPage="currentPage" :totalPages="20" />
</template>
```

With the URL `?page=5`, the component will emit `update:currentPage` with `5` on mount. Clicking page 3 updates the URL to `?page=3`.

### Custom query parameter

```vue
<template>
  <FzPagination v-model:currentPage="currentPage" :totalPages="20" urlKey="p" />
</template>
```

### Multiple paginations

Assign distinct `urlKey` values to avoid conflicts:

```vue
<template>
  <FzPagination v-model:currentPage="itemsPage" :totalPages="20" urlKey="itemsPage" />
  <FzPagination v-model:currentPage="commentsPage" :totalPages="5" urlKey="commentsPage" />
</template>
```

### Disabling URL sync

```vue
<template>
  <FzPagination v-model:currentPage="currentPage" :totalPages="20" :syncUrl="false" />
</template>
```

## Custom UI with usePagination

If you need a fully custom pagination UI but want to reuse the page calculation logic (clamping, ellipsis, anchors, prev/next), import the `usePagination` composable directly. It returns a reactive list of `PaginationItem` objects that you can render however you like.

```vue
<script setup>
import { usePagination } from '@fiscozen/pagination'
import { ref } from 'vue'

const currentPage = ref(1)
const totalPages = 20

const { items } = usePagination(currentPage, totalPages, {
  firstPage: { show: true },
  lastPage: { show: true },
  prev: { label: 'Prev' },
  next: { label: 'Next' }
})

const goToPage = (page) => {
  currentPage.value = page
}
</script>

<template>
    <nav aria-label="Paginazione">
    <ul class="flex gap-2">
      <li v-for="(item, i) in items" :key="`${item.type}-${item.value}-${i}`">
        <span v-if="item.type === 'ellipsis'" class="px-2">...</span>
        <button
          v-else
          :disabled="item.disabled"
          :aria-current="item.current ? 'page' : undefined"
          class="px-3 py-1 rounded"
          :class="item.current ? 'bg-blue-500 text-white' : 'bg-gray-100'"
          @click="goToPage(item.value)"
        >
          {{ item.label }}
        </button>
      </li>
    </ul>
  </nav>
</template>
```

## Accessibility

- Uses semantic `nav` element with `aria-label="Paginazione"`
- Keyboard navigable controls (Tab, Enter, Space)
- Disabled state on prev/next buttons at boundaries
- Screen reader compatible button labels

## Best Practices

- Always bind `v-model:currentPage` to a reactive `ref` for two-way page tracking
- The component hides itself when `totalPages` is 1 or less
