# FzPagination

Page-based navigation control for paginated data sets. Provides intuitive page navigation with support for two-way binding, configurable labels, ellipsis handling, and responsive behavior.

## Features

- Two-way binding via v-model:currentPage for current page
- Configurable total number of pages
- Responsive layout (icon-only prev/next on mobile)
- Accessible navigation with semantic HTML and ARIA attributes
- Frontoffice and backoffice environment support

## Installation

```bash
npm install @fiscozen/pagination
```

## Basic Usage

```vue
<script setup>
import { FzPagination } from '@fiscozen/pagination'
import { ref } from 'vue'

const currentPage = ref(1)
</script>

<template>
  <FzPagination v-model:currentPage="currentPage" :total-pages="10" />
</template>
```

## Custom UI with usePagination

If you need a fully custom pagination UI but want to reuse the page calculation logic (clamping, ellipsis, anchors, prev/next), import the `usePagination` composable directly. It returns a reactive list of `PaginationItem` objects that you can render however you like.

```vue
<script setup>
import { usePagination } from '@fiscozen/pagination'
import { ref } from 'vue'

const currentPage = ref(1)
const totalPages = 20

const { items } = usePagination(currentPage, totalPages, {
  firstPage: { show: true },
  lastPage: { show: true },
  prev: { label: 'Prev' },
  next: { label: 'Next' }
})

const goToPage = (page) => {
  currentPage.value = page
}
</script>

<template>
    <nav aria-label="Paginazione">
    <ul class="flex gap-2">
      <li v-for="(item, i) in items" :key="`${item.type}-${item.value}-${i}`">
        <span v-if="item.type === 'ellipsis'" class="px-2">...</span>
        <button
          v-else
          :disabled="item.disabled"
          :aria-current="item.current ? 'page' : undefined"
          class="px-3 py-1 rounded"
          :class="item.current ? 'bg-blue-500 text-white' : 'bg-gray-100'"
          @click="goToPage(item.value)"
        >
          {{ item.label }}
        </button>
      </li>
    </ul>
  </nav>
</template>
```

## Accessibility

- Uses semantic `nav` element with `aria-label="Paginazione"`
- Keyboard navigable controls (Tab, Enter, Space)
- Disabled state on prev/next buttons at boundaries
- Screen reader compatible button labels

## Best Practices

- Always bind `v-model:currentPage` to a reactive `ref` for two-way page tracking
- The component hides itself when `totalPages` is 1 or less
