# FzTextarea

Multi-line text input component with label, validation states, resize control, and full WCAG 2.1 AA accessibility support.

## Features

- Configurable text sizes (sm, md, lg)
- Resize control (none, vertical, horizontal, all)
- Error and valid states with visual feedback
- Help text and error messages
- Required, disabled, and readonly states
- Full ARIA accessibility (aria-required, aria-invalid, aria-disabled, aria-describedby)
- Error messages announced via role="alert"
- Auto-generated IDs for label association

## Installation

```bash
npm install @fiscozen/textarea
```

## Basic Usage

```vue
<script setup>
import { FzTextarea } from '@fiscozen/textarea'
import { ref } from 'vue'

const notes = ref('')
</script>

<template>
  <FzTextarea label="Notes" v-model="notes" />
</template>
```

## Examples

### With Validation

```vue
<script setup>
import { FzTextarea } from '@fiscozen/textarea'
import { ref, computed } from 'vue'

const description = ref('')
const hasError = computed(() => description.value.length > 0 && description.value.length < 10)
</script>

<template>
  <FzTextarea
    label="Description"
    v-model="description"
    :error="hasError"
    errorMessage="Description must be at least 10 characters"
    required
  />
</template>
```

### With Help Text

```vue
<template>
  <FzTextarea
    label="Bio"
    v-model="bio"
    helpMessage="Tell us about yourself (max 500 characters)"
    :maxlength="500"
    resize="vertical"
  />
</template>
```

### Disabled State

```vue
<template>
  <FzTextarea
    label="Locked Field"
    modelValue="This content cannot be edited"
    disabled
  />
</template>
```

### Readonly State

```vue
<template>
  <FzTextarea
    label="Terms and Conditions"
    modelValue="Lorem ipsum dolor sit amet..."
    readonly
    :rows="6"
  />
</template>
```

### Custom Size and Resize

```vue
<template>
  <FzTextarea
    label="Small, vertical-only resize"
    v-model="text"
    size="sm"
    resize="vertical"
    :rows="4"
  />
</template>
```

## Accessibility

FzTextarea meets WCAG 2.1 AA standards:

- **ARIA attributes**: `aria-required`, `aria-invalid`, `aria-disabled` always present with string values ("true"/"false") for screen reader compatibility
- **aria-describedby**: Links textarea to error or help message when present
- **role="alert"**: Error messages are announced immediately by screen readers
- **Label association**: Label is always associated with textarea via `for`/`id` binding (auto-generated ID fallback)
- **Decorative icons**: Check and error icons have `aria-hidden="true"`
- **Keyboard navigation**: Full keyboard support (Tab to focus, type to input)

## Best Practices

- Always provide a descriptive `label` for accessibility
- Use `helpMessage` to guide users on expected input format
- Use `errorMessage` paired with `error` prop for validation feedback
- Set `resize="vertical"` for most use cases to prevent horizontal overflow
- Use `rows` to set a sensible initial height based on expected content length
- Use `maxlength` to enforce content limits
