# FzTextarea

Multi-line text input component with label, validation states, resize control, and full WCAG 2.1 AA accessibility support.

## Features

- Resize control (none, vertical, horizontal, all)
- Error and valid states with visual feedback
- Error messages via FzAlert component (circle-xmark icon, consistent with FzInput)
- Custom focus styling (blue border on focus, error border maintained on error+focus)
- Help text and error messages with multiline wrapping support
- Required, disabled, and readonly states with proper visual cues
- Combined states support (error+disabled, required+help, disabled+compiled)
- Full ARIA accessibility (aria-required, aria-invalid, aria-disabled, aria-labelledby, aria-describedby)
- Error messages announced via role="alert"
- Auto-generated IDs for label association
- Exposed `textareaRef` for programmatic focus
- Native event forwarding via `v-bind="$attrs"` (inheritAttrs: false)

## Installation

```bash
npm install @fiscozen/textarea
```

## Basic Usage

```vue
<script setup>
import { FzTextarea } from '@fiscozen/textarea'
import { ref } from 'vue'

const notes = ref('')
</script>

<template>
  <FzTextarea label="Notes" v-model="notes" />
</template>
```

All native DOM events (<code>blur</code>, <code>focus</code>, <code>paste</code>, <code>keydown</code>, etc.) are automatically forwarded to the native <code>&lt;textarea&gt;</code> element via <code>v-bind="$attrs"</code>. You can listen to any native event directly on the component:

```vue
<template>
  <FzTextarea label="Notes" @blur="onBlur" @keydown.enter="onEnter" />
</template>
```

Any extra HTML attributes (e.g. <code>data-*</code>, <code>class</code>, <code>style</code>) are also applied to the native textarea, not the wrapper div.

## Examples

### With Validation

```vue
<script setup>
import { FzTextarea } from '@fiscozen/textarea'
import { ref, computed } from 'vue'

const description = ref('')
const hasError = computed(() => description.value.length > 0 && description.value.length < 10)
</script>

<template>
  <FzTextarea
    label="Description"
    v-model="description"
    :error="hasError"
    required
  >
    <template #errorMessage>Description must be at least 10 characters</template>
  </FzTextarea>
</template>
```

### With Help Text

```vue
<template>
  <FzTextarea
    label="Bio"
    v-model="bio"
    :maxlength="500"
    resize="vertical"
  >
    <template #helpText>Tell us about yourself (max 500 characters)</template>
  </FzTextarea>
</template>
```

### Disabled State

```vue
<template>

  />
```

## Accessibility

FzTextarea meets WCAG 2.1 AA standards:

- **Focus indicator**: Custom blue border (`blue-600`) on focus replaces browser default outline. Error state maintains error border color on focus (`semantic-error-300`).
- **ARIA attributes**: `aria-required`, `aria-invalid`, `aria-disabled` always present with string values ("true"/"false") for screen reader compatibility
- **aria-describedby**: Links textarea to error or help message when present
- **role="alert"**: Error messages are announced immediately by screen readers
- **aria-labelledby**: References the label element ID for stronger screen reader association
- **Label association**: Label is always associated with textarea via `for`/`id` binding and `aria-labelledby` (auto-generated ID fallback)
- **Decorative icons**: Check icon and FzAlert error icon have `aria-hidden="true"`
- **Combined states**: Error+disabled, required+help, and other combinations maintain correct ARIA semantics
- **Keyboard navigation**: Full keyboard support (Tab to focus, type to input)

## Best Practices

- Always provide a descriptive `label` for accessibility
- Use `helpText` slot to guide users on expected input format
- Use `errorMessage` slot paired with `error` prop for validation feedback
- Combine `required` with `helpText` slot to explain what the field needs
- Set `resize="vertical"` for most use cases to prevent horizontal overflow
- Use `rows` to set a sensible initial height based on expected content length
- Use `maxlength` to enforce content limits
- When showing pre-filled disabled fields, ensure the value conveys why the field is locked
