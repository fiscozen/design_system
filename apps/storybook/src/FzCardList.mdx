# FzCardList

A container that renders a vertical list of `FzCardListItem` rows from the required `items` array. Use your own heading and surrounding layout (for example an `h2` and filters) beside or above the list—`FzCardList` does not render a section title or accept slots.

## Features

- Data-driven list via the `items` prop (each entry maps to one row)
- Optional badge, value, descriptions, indicator icon, and row actions (arrow or overflow menu)
- `fzaction:click` on the list includes the item index plus the action index and action payload

## Installation

```bash
npm install @fiscozen/card-list
```

## Basic Usage

```vue
<script setup>
import { FzCardList } from '@fiscozen/card-list'

const items = [
  {
    title: 'Fattura #001',
    value: '1.200,00 €',
    badge: { text: 'Bozza', tone: 'dark' },
    showIndicator: true,
    descriptions: ['Cliente: Rossi S.r.l.'],
    actions: [{ type: 'action', variant: 'textLeft', label: 'Apri' }]
  },
  {
    title: 'Fattura #002',
    value: '3.450,00 €',
    showIndicator: false,
    descriptions: ['Cliente: Bianchi S.p.A.']
  }
]

function onRowAction(itemIndex, actionIndex, action) {
  console.log(itemIndex, actionIndex, action)
}
</script>

<template>
  <div>
    <h2 class="m-0">Fatture</h2>
    <FzCardList :items="items" @fzaction:click="onRowAction" />
  </div>
</template>
```

### Using FzCardListItem without FzCardList

`FzCardList` has no default slot. Compose `FzCardListItem` yourself when you need custom wrappers or non-uniform rows.

```vue
<script setup>
import { FzCardListItem } from '@fiscozen/card-list'
</script>

<template>
  <div>
    <FzCardListItem
      title="Voce personalizzata"
      value="200,00 €"
      :badge="{ text: 'Custom', tone: 'dark' }"
      :show-indicator="true"
      :descriptions="['Nota: verifica allegati']"
      :actions="[{ type: 'action', variant: 'textLeft', label: 'Apri' }]"
    />
  </div>
</template>
```

### Indicator on or off

```vue
<script setup>
import { FzCardList } from '@fiscozen/card-list'

const items = [
  { title: 'With indicator', showIndicator: true, descriptions: ['Row A'] },
  { title: 'Without indicator', showIndicator: false, descriptions: ['Row B'] }
]
</script>

<template>
  <FzCardList :items="items" />
</template>
```

## Accessibility

- Provide a visible section title in the parent layout (for example an `h2`) so the list sits in a clear heading structure.
- Row actions use an icon button with `aria-label` “Open” (single action) or an overflow control with “Open menu” (multiple actions); keyboard users can Tab to these controls.
- Title text is truncated with a single-line paragraph so long labels stay usable in narrow layouts.

## Best Practices

- Keep `items` as the source of truth for homogeneous lists (API results, tables of cards).
- Use one action for a primary navigation or command; use several when you need a compact overflow menu.
- Handle `fzaction:click` using `itemIndex` to map back to your data row.
- Import `FzCardListItem` when you need slots or mixed content around each row; `FzCardList` is only for the data-driven array pattern.

## Related Components

- **FzCardListItem**: Single row; same props as each `items` entry; also emits `fzaction:click` and `update:isOpen` when used standalone.
- **FzAction / FzIconDropdown**: Underlying action model for `actions`.
