# useQueryString

Type-safe URL query parameter management with History API synchronization. Bridges the gap between Vue Router's read-only `route.query` and the need to read/write typed query params directly.

## Features

- Type-safe extraction with automatic transformation (string, number, boolean, custom)
- Default values for missing or invalid parameters
- History API synchronization (replaceState / pushState)
- Vue Router integration via provide/inject (optional)
- Router-agnostic mode for non-Vue-Router environments
- History state persistence to survive Vue Router navigation quirks

## Installation

```bash
npm install @fiscozen/composables
```

## Basic Usage

```vue
<script setup>
import { useQueryString } from '@fiscozen/composables'

const { initialValuesInQueryString, setValuesInQueryString } = useQueryString([
  { key: 'page', defaultValue: 1, transform: 'number' },
  { key: 'search', defaultValue: '', transform: 'string' },
])

// Read initial values (frozen at mount time)
console.log(initialValuesInQueryString.page)   // number
console.log(initialValuesInQueryString.search)  // string

// Write values to the URL
setValuesInQueryString({ page: 2, search: 'hello' })
</script>
```

## Vue Router Integration

For apps using Vue Router, provide the route once at app level so all consumers can access it without prop drilling:

```vue
<script setup>
// App.vue (once)
import { provideQueryStringRoute } from '@fiscozen/composables'
import { useRoute } from 'vue-router'

provideQueryStringRoute(useRoute())
</script>
```

Then any component can call `useQueryString` without passing a route:

```ts
const { initialValuesInQueryString, setValuesInQueryString } = useQueryString([
  { key: 'tab', defaultValue: 'overview', transform: 'string' },
])
```

The composable auto-injects the provided route. If no provider exists, it falls back to router-agnostic mode automatically.

### Explicit Route

You can pass a specific route instance directly:

```ts
import { useRoute } from 'vue-router'

const route = useRoute()
const { ... } = useQueryString([...], route)
```

### Force Router-Agnostic Mode

Pass `null` to bypass Vue Router entirely, even when a provider exists:

```ts
const { ... } = useQueryString([...], null)
```

## Key Configuration

Each key can be a simple string or a configuration object:

### Simple String Key

```ts
useQueryString(['page', 'search'])
```

Reads the raw string value as-is; no transformation or default.

### Configuration Object

```ts
useQueryString([
  {
    key: 'page',
    defaultValue: 1,
    transform: 'number',
  },
  {
    key: 'active',
    defaultValue: true,
    transform: 'boolean',
  },
  {
    key: 'filter',
    defaultValue: null,
    transform: 'string',
    nullStringIsNullValue: true,
  },
])
```

## Examples

### Paginated Table

```vue
<script setup>
import { ref, watch } from 'vue'
import { useQueryString } from '@fiscozen/composables'

const { initialValuesInQueryString, setValuesInQueryString } = useQueryString([
  { key: 'page', defaultValue: 1, transform: 'number' },
  { key: 'perPage', defaultValue: 25, transform: 'number' },
])

const page = ref(initialValuesInQueryString.page)
const perPage = ref(initialValuesInQueryString.perPage)

watch([page, perPage], ([p, pp]) => {
  setValuesInQueryString({ page: p, perPage: pp })
  fetchData(p, pp)
})
</script>
```

### Search with Filters

```vue
<script setup>
import { useQueryString } from '@fiscozen/composables'

const { initialValuesInQueryString, setValuesInQueryString } = useQueryString([
  { key: 'q', defaultValue: '', transform: 'string' },
  { key: 'category', defaultValue: 'all', transform: 'string' },
  { key: 'active', defaultValue: true, transform: 'boolean' },
])
</script>
```

### Custom Transform Function

```ts
const { initialValuesInQueryString } = useQueryString([
  {
    key: 'tags',
    defaultValue: [],
    transform: (value) => (typeof value === 'string' ? value.split(',') : []),
  },
])
// ?tags=vue,react → initialValuesInQueryString.tags === ['vue', 'react']
```

## Accessibility

This composable does not render UI and has no direct accessibility implications. Components that use it (like FzPagination) are responsible for accessible rendering.

## Best Practices

- Call `provideQueryStringRoute(useRoute())` once in App.vue for Vue Router apps
- Use `transform: 'number'` for pagination and numeric filters to avoid string comparisons
- Set meaningful `defaultValue` for every key to handle missing params gracefully
- Use `getInitialPageFromUrl` from `@fiscozen/pagination` for the specific case of initializing a page ref synchronously in `setup()`
